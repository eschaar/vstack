# Release 2026-04-26 — v2.0.0

## Summary

vstack 2.0.0 replaces the monolithic `commands.py` CLI module with a clean,
module-per-command architecture and extracts manifest persistence into its own
`vstack.manifest` package. Import paths for CLI classes have changed — this is a
breaking release. The `vstack manifest upgrade --backfill` command is now available
for retroactively adding checksums to legacy manifest entries.

## Breaking changes

- `vstack.cli.commands` is removed. `CommandService` and command classes now live in
  dedicated modules under `vstack.cli` (`service`, `install`, `verify`, `validate`,
  `status`, `uninstall`, `manifest`, `report`).
- Manifest persistence classes (`Manifest`, `ManifestFile`, `ArtifactEntry`, etc.)
  have moved to the new `vstack.manifest` package (`vstack.manifest.store`).
- Any code that imported directly from `vstack.cli.commands` must update its imports.

## What's new

- `vstack manifest upgrade --backfill`: retroactively compute and write checksums for
  tracked manifest entries that have no checksum (useful after migrating from legacy
  manifests).
- CLI architecture split across 12 focused modules: `base`, `catalog`, `helpers`,
  `registry`, `service`, `interface`, `install`, `verify`, `validate`, `status`,
  `uninstall`, `report` — each independently testable and importable.
- New `vstack.manifest` package with `Manifest`, `ManifestFile`, `ArtifactEntry`,
  `content_hash`, and schema-versioned read/write logic.

## Fixed

- `_version_gt` in `InstallCommand` no longer passes `None` when `existing` is absent;
  falls back to `"0"` for comparison, eliminating a `TypeError` on first install.

## Internal

- Full test suite restructured: one test file per source module, `TestClass` per class,
  `test_function` per standalone function. Catch-all files (`test_commands.py`,
  `test_coverage_blockers.py`) deleted. Test count: 288 → 337.
- mypy added to the CI quality gate — 106 files, 0 errors.
- End-to-end integration tests consolidated into `tests/vstack/test_integration.py`.
- 4 new ADRs documenting manifest schema versioning (014), conservative install
  defaults (015), atomic manifest writes (016), and checksum backfill (017).
